# Повторяем базу

# SOLID - https://en.wikipedia.org/wiki/SOLID

Single responsibility - у типа или пакета должна быть одна причина для изменения - одна зона ответственности.

Open–closed - код открыт для расширения, но закрыт для модификации. Новое поведение - это новый тип, а не правка существующего

Liskov substitution - любая реализация интерфейса должна вести себя так, чтобы её можно было подставить вместо другой, не ломая ожидания вызывающего кода. Реализация не должна нарушать «контракт» интерфейса.

Interface segregation - много маленьких специализированных интерфейсов лучше одного большого. Клиент не должен зависеть от методов, которые он не использует.


Dependency inversion - модули верхнего уровня (бизнес-логика) не должны зависеть от модулей нижнего уровня (БД, HTTP, файлы). И те и другие должны зависеть от абстракций.
